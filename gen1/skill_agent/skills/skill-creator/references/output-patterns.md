## JSON Output Pattern

Use when the output will be consumed programmatically:

```markdown
## Output Format

Return a JSON object:
```json
{
  "status": "success | error",
  "result": { ... },
  "message": "human-readable summary"
}
```

Always return valid JSON. Do not wrap in markdown code fences unless the user asks.

```

## Step-by-Step Output Pattern

Use for instructional or procedural outputs:

```markdown
## Output Format

Return numbered steps with this structure:

### Step {N}: {Title}
**Action**: {what to do}  
**Command** (if applicable):
```bash
{command}
```

**Expected result**: {what success looks like}
**If this fails**: {fallback or error guidance}

```

## Rules for Output Formats

- Define the output format with a concrete example, not just a description
- Use the exact headings the agent should produce â€” not placeholders like `{section}`
- If the output can vary by case, show the most common variant and note variations
- Specify edge cases: what to return if input is empty, invalid, or ambiguous
